# 设计原则（Designing Principle）

## 定义

设计原则是将故事组织为一个整体的**内在逻辑**，是故事具有独创性和高效性的最重要因素。公式：**设计原则 = 故事发展过程 + 独创性的手法**。

设计原则与故事前提的区别：
- **故事前提**：具体的，是实际发生的事（"一个黑手党家庭的小儿子向射杀父亲的人复仇"）
- **设计原则**：抽象的，是运行于故事深层的过程（"运用经典童话策略，呈现三个儿子中最小的儿子如何成为新君王"）

设计原则可以从以下途径产生：象征/隐喻、时间单位、旅程结构、独特的叙事者、故事展开的特殊方式。

## 判定标准

- 是否能用**一句话**表述故事的整体策略
- 是否揭示了故事的**独创性**所在（而非类型通用公式）
- 是否体现了故事**发展的过程**（而非只是一个概念）
- 是否让故事"整体格局大于部分之和"
- 好故事必有设计原则——大部分平庸故事只有前提没有设计原则

## 常见错误

1. **用类型替代设计原则**：把"这是个爱情故事"或"这是个侦探故事"当做设计原则。类型是预先设定的框架，设计原则是独特的组织逻辑。
2. **从外部强加而非从前提中归纳**：设计原则隐含在故事前提之中，需要像侦探一样"归纳"出来。
3. **设计原则与前提混为一谈**：设计原则是抽象的"如何"，前提是具体的"什么"。
4. **忽视设计原则的存在**：直接写故事而不先明确设计原则，容易落入公式化。
5. **中途放弃设计原则**：设计原则应作为整部作品的指南针，时刻紧盯着它。

## 交叉引用：McKee 控制意念（Controlling Idea）

| 维度 | Truby 设计原则 | McKee 控制意念 |
|------|---------------|---------------|
| 本质 | 故事的形式和过程（"如何"） | 故事的价值核心（"为什么"） |
| 功能 | 组织整体结构，确定独创性 | 提供主题价值观的方向 |
| 表述方式 | "运用X策略，呈现Y的发展过程" | "正义终将战胜邪恶"（价值观陈述） |
| 角色 | 故事的种子和蓝图 | 故事的道德指南针 |
| 与主题关系 | 从设计原则衍生主题要旨 | 本身就是主题陈述 |
| 适用场景 | 构思阶段的策略工具 | 修改阶段的统一性原则 |

**互补使用**：先用Truby的设计原则确定故事独特的发展形式，再用McKee的控制意念确保价值观的清晰和一致性。
