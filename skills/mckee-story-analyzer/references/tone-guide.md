# 语气输出规范 (Tone Guide)

## 概述
本文件定义了 mckee-story-analyzer 的三种分析语气模式。在每次分析开始前，用户将通过选择界面指定语气。以下规范给出了每种语气在措辞、结构、情绪基调上的具体规则。

## 三种语气对比

### 一、尖刻模式 (Sharp Mode)

**核心定位：** 直截了当地指出缺陷并提出改进意见。以最有效率的方式传达核心问题。
**适用场景：** 已有一定基础的创作者、需要快速找到致命问题的审读、能承受直白批评的专业人士。

**措辞规则：**
- 跳过"先扬后批"：不需要先铺垫优点
- 使用诊断式语言："问题是..."、"这不行因为..."、"无效"
- 不保留意见：直接说出判定结果
- 引用理论时不解释理论：默认读者知道
- 修改建议直接给出，不加"可能"、"也许"

**禁止的用语：**
- "整体来说还不错"类的笼统安慰
- "有潜力"（不说"有潜力"，直接说"需要什么"）
- "这里有一些小建议"（暗示问题不大）

**鼓励的用语：**
- "问题出在..."
- "这段不成立因为..."
- "直接改：..."
- "诊断：[具体问题]"

**输出结构：**
```
## 诊断报告

### 致命问题（★★★）
[直接指出最严重的问题，每条一针见血]

### 次要问题（★★☆）
[...]

### 结论
[1-2句核心修改方向]
```

**示范对照：**

> **客观模式会写：** "你的激励事件设定在第十五页，这个位置总体来说是合理的。不过它目前的力度稍显不足，可以考虑加重主人公的个人牵涉。"
> 
> **尖刻模式写：** "激励事件在第十五页——可以。但力度不够。主人公面对这个事件，完全可以合理地说'我不参与'而生活不受影响。你需要让他没有选择余地。现在这个激励是软的。"

### 二、客观模式 (Objective Mode) [默认]

**核心定位：** 同时客观地指出作品的缺点和长处，提出改进意见。专业持平，不偏不倚。
**适用场景：** 希望获得全面、平衡评估的创作者、竞赛作品评估、出版审读。

**措辞规则：**
- 严格遵守"先扬后批"：每个维度先肯定优点，再指出问题
- 使用专业、距离感的措辞
- 所有诊断必须有理论依据引用
- 优缺点比例大致保持 1:1
- "问题"和"优点"同等篇幅

**输出结构：**
```
## 综合评估

### 核心优势
1. [具体优势 + 理论依据]
2. ...

### 核心问题（按优先级排序）
1. [问题] — 严重程度 ★★★ — 影响范围：...
2. [问题] — 严重程度 ★★☆ — 影响范围：...

### 维度分析

#### [维度名称]
**优点：** [与麦基理论的契合点]
**不足：** [与麦基理论的偏离点]
**建议：** [可操作的修改方向]
```

**示范：**
> "在故事结构方面，你的激励事件在第十七页出现，这个位置总体上合理——根据麦基的框架，激励事件应在前 1/4 处触发，而你的节奏控制做得不错。不过，在两个方面可以加强：一是激励事件的个人关联度不足，这让主人公的介入动机显得被动；二是鸿沟在后续序列中的加深不够明显——第二层和第三层鸿沟在严重程度上差别不够大，导致故事在中段有一定的张力平坡。"

### 三、鼓励模式 (Encouraging Mode)

**核心定位：** 语气温和积极，问题包裹在肯定中提出。以建设性为导向，避免让创作者感到挫败。
**适用场景：** 新手作者、敏感期的创作者、需要建立信心的写作阶段。

**措辞规则：**
- 强制"正面先行"：每个诊断段落必须以肯定开头
- 问题的表述方式："这里有一个可以让它更强大的方向..."
- 肯定/建议比例：2:1 或 3:1
- 建议用"可以试试"而非"应该"、"必须"
- 即使是最弱的部分也先找到肯定点

**禁止的用语：**
- "这不行"
- "严重缺陷"
- "缺乏"
- "失败"

**鼓励的用语：**
- "这里有一特别有意思的东西，如果再..."
- "一个有意思的方向是..."
- "你已经做到了X，而如果进一步..."
- "这已经有很好的基础了..."

**输出结构：**
```
## 整体感受

[1-2段对作品整体的正向评价]

## 这里有一些方向可以让它更亮眼

### [维度名称]
**这里做得很好：** [具体优势，充分展开]

**一个可以继续探索的方向：**
[问题包裹在正向语言中，作为"可探索的方向"而非"需要修复的缺陷"]

**试试看这样：**
[示范改写或结构调整建议，以"一个可能的版本是..."开头]
```

**示范对照：**

> **客观模式会写：** "你的主人公缺少了一个人物弧光——在整个故事中，他从头到尾没有发生任何内在转变，这使他显得平面。"
> 
> **鼓励模式写：** "你在主人公的塑造上花费了明显的心思——他的语言风格前后一致，读者能很快建立起对这个人的熟悉感。一个可以让这个人物在读者心中走得更远的方向是，考虑在故事的某个关键节点——比如在他做出那个困难选择的时候——加入一个细微的内在转变。这不需要是一个巨大的人物弧光，一个小小的、只有读者能注意到的'变化'就能带来很强的共鸣感。"

## 三种语气的快速判断矩阵

| 特征 | 尖刻模式 | 客观模式 | 鼓励模式 |
|------|---------|---------|---------|
| 先扬后批 | 否 | 是 | 必须 |
| 直接批评 | 大量 | 平衡 | 极少 |
| 理论引用 | 简洁 | 具体 | 作为背景不喧宾夺主 |
| 示范改写 | 必有 | 必有 | 以"一个可能的版本"引入 |
| 治疗比喻 | 外科手术 | 体检报告 | 营养师建议 |
| 对话语气 | 直白 | 专业 | 温和 |
| 严重程度标注 | 用 | 用 | 改用"影响范围"或其他温和表达 |
| 输出长度 | 最短 | 标准 | 最长（因为需要铺开肯定部分） |

## 三种语气在 Step 1 时的使用

当用户提交作品进行初始分类后，主 skill 将弹出以下选择界面：

```
请选择分析语气：

[A] 尖刻模式 — 直截了当，不绕弯子。适合已经有了基础、想找致命问题的创作者。
[B] 客观模式 — 优缺点并重，专业持平。适合希望获得全面评估的创作者。[默认]
[C] 鼓励模式 — 语气温和，以建设为导向。适合新手或需要信心支持的创作者。
```

选择后，整个分析将按照所选语气的规则生成输出。
