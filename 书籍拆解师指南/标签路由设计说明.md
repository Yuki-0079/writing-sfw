# 标签路由系统设计说明

> 本文档解释知识体系 v2 中标签路由系统的设计动机、工作原理和扩展方法。

## 为什么需要标签路由？

在 v1 架构中，Writing Analyzer 通过硬编码规则做作品的类型判断后匹配固定知识源。这种方式的局限：

- **路径耦合**：新增一个知识源必须修改 Analyzer 本体
- **类型僵化**：写赛博朋克时可能既需要世界观构建（经济体系）也需要写作技法（人物）
- **跨域盲区**：无法感知政治经济学知识库对反乌托邦创作的辅助价值

v2 标签路由将匹配逻辑从 Analyzer 内部拆出到 `knowledge-registry.json`，实现了 **用户需求 → 标签推理 → 注册表匹配 → 按需加载** 的解耦链路。

## 四维度设计

### 为什么是这四个维度？

| 维度 | 解决的问题 | 类比 |
|------|-----------|------|
| `craft_*` | 怎么写？ | 语法+n |
| `world_*` | 写在哪？ | 语境 |
| `domain_*` | 凭什么？ | 理论依据 |
| `genre_*` | 写给谁？ | 读者预期 |

四个维度覆盖了创作者在构思和落笔时面临的所有决策层。单一维度不够——你可能在写一篇赛博朋克（genre）小说，需要设计经济体系（world），同时使用多视角叙事技巧（craft），还要参考马克思的批判理论（domain）。

### 标签粒度的选择

每个维度下细分 9-12 个标签，控制在"一眼扫完"的粒度。过细则管理成本高；过粗则失去路由精度。

### 标签前缀的命名惯例

`维度_领域` 格式（如 `craft_structure`、`world_economy`）：
- 前缀保证标签在全文搜索中的唯一性
- 下划线后的领域名直观可读
- 支持正则通配匹配（如 `craft_*` 匹配所有写作技法）

## 推理机制

### 三步推理法

```
Step 1: 类型语境（genre_*）→ 确定基本体裁约定
  用户说"写一个赛博朋克故事" → genre_cyberpunk
  用户说"帮我打磨这段对白" → 无明确类型，跳过

Step 2: 领域需求（domain_* / world_*）→ 确定需要的深度知识
  用户说"企业取代了政府" → world_politics, world_economy
  用户说"外星文明的宗教" → world_culture, genre_sf

Step 3: 写作技法（craft_*）→ 确定叙事层面的指导
  用户说"这段对话很木" → craft_dialogue
  用户说"开场不够吸引人" → craft_structure, craft_pacing
```

### 关键词 → 标签映射表

完整的推理映射定义在 `memory/knowledge-tag-taxonomy.md` 第 6 节。示例：

| 关键词 | 推理标签 |
|--------|---------|
| 结构、大纲、三幕 | craft_structure |
| 经济、货币、贸易 | world_economy, domain_economics |
| 赛博、朋克、义体 | genre_cyberpunk |
| 社会、阶级 | world_society, domain_sociology |

### 综合优先级

```
genre_*（体裁语境）→ world_*/domain_*（领域知识）→ craft_*（写作技法）
```

## 注册表匹配

`knowledge-registry.json` 中每个知识源通过 `tags` 字段声明自己的标签集合：

```json
{
  "slug": "mckee-story-analyzer",
  "tags": ["craft_structure", "craft_character", "craft_scene", "craft_dialogue"],
  "relevance": { "craft": "very_high", "worldbuilding": "low", "theory": "high" }
}
```

匹配规则：
1. 取用户推理出的标签与知识源标签的交集
2. 交集最大的优先
3. 交集相同时用 `relevance` 评分加权
4. 只加载 Top-3 知识源（节省上下文）

## 如何扩展

### 新增一个知识源

1. 在 `knowledge-registry.json` 的 `skills` 数组中添加新条目
2. 从三个维度各选 1-3 个标签
3. 可选：在 `knowledge-tag-taxonomy.md` 中追加新的推理映射

### 新增一个标签

如果现有标签不足以覆盖新知识源：

1. 确定属于哪个维度（craft_* / world_* / domain_* / genre_*）
2. 在 `knowledge-tag-taxonomy.md` 对应维度的表格中追加一行
3. 在 `关键词→标签映射表` 中加入新的推理规则
4. 为新知识源打上标签

### 修改推理规则

`knowledge-tag-taxonomy.md` 第 6 节的映射表是 Analyzer 的推理字典。修改映射表即可调整路由行为，无需修改 Analyzer 本体。
