# 书籍知识库构建流程规范 v1

> 将创意写作类书籍转化为结构化知识库 skill 的标准化流程。
> 参考：mckee-story-analyzer（知识库模式）+ thinking-knowledge-system（方法论）+ karpathy-llm-wiki（运维规范）

---

## 一、概述

本流程规范定义了将一本创意写作书籍转化为可复用的 OpenClaw skill 知识库的标准操作程序。

### 产出物

每本书最终的 skill 目录结构：

```
《书籍名》知识库/
├── SKILL.md                          ← 技能主指令文件
├── references/
│   ├── knowledge-base.json           ← 结构化知识库（核心资产）
│   ├── toc-index.json                ← 章节索引
│   ├── chapters/                     ← 原文分章文件
│   │   ├── 0001.txt
│   │   ├── 0002.txt
│   │   └── ...
│   ├── deep/                         ← 专题深度分析文档
│   │   ├── concept1.md
│   │   └── ...
│   └── case-studies/ (可选)           ← 应用案例分析
├── scripts/
│   ├── query_kb.ps1                  ← 知识库查询脚本
│   └── (extract_kb.ps1)              ← 构建脚本
└── _meta.json
```

### 输入

- 一本书的文本格式（epub / pdf / markdown）

---

## 二、转化流水线：四阶段

### 阶段 1：文本提取与分章

**目标**：将原始书籍转为可处理的纯文本，按自然章节拆分。

**步骤**：
1. epub → 提取为纯文本/Markdown
2. PDF → 提取文本，对照校正
3. 清洗：去除页眉页脚、出版信息、索引等非正文内容
4. 按章节拆分文件，命名为 `references/chapters/NNNN.txt`（4位数字编号）
5. 生成 `references/toc-index.json`：记录每个章节的文件名和标题

**输出**：
- `references/chapters/0001.txt` ~ `NNNN.txt`
- `references/toc-index.json`

### 阶段 2：知识提取与概念标注

**目标**：逐章精读，提取可复用知识点，标注概念标签。

**阅读规范**：在提取前必须加载 `close-reading-protocol` 按逐块精读流程执行。每章内容必须完整阅读（禁止跳读首尾），提取的知识条目须注明来源章节和原文位置。

**方法（参考 thinking-knowledge-system 的拆→归→联→提）**：

对每一章：
1. **拆分**：将章节内容按语义单元切分（每个单元是一个完整论述）
2. **归纳**：提取核心论点、方法、案例、定义
3. **联系**：标注与书中其他章节的关联概念
4. **提炼**：将描述性文字转述为可执行的方法论

**知识条目 JSON 格式（Schema v2 - 通用领域）**：

```json
{
  "id": "book-slug-001",
  "domain": "writing_craft | political_economy | worldbuilding | sociology | ...",

  "chapter": "章节标题",
  "heading": "小节标题",
  "concepts": ["concept1", "concept2"],
  "text": "提炼后的知识单元（非全文照搬）",

  "source": {
    "chapter_file": "chapters/0001.txt",
    "page_range": "45-48"
  },

  "confidence": "high | medium | low | contested",
  "scope_note": "适用范围和条件（如果有）",

  "fiction_application": "这个知识在小说/世界观构建中的用法（可选）",

  "methodology": {
    "steps": ["步骤1", "步骤2"],
    "key_points": ["关键点"]
  }
}
```

**字段说明**：

| 字段 | 必填 | 说明 |
|------|:----:|------|
| `id` | ✅ | 唯一标识：`{书slug}-{序号}` |
| `domain` | ✅ | 知识领域，见下方域值表 |
| `chapter` | ✅ | 原文章节标题 |
| `heading` | ✅ | 原文小节标题 |
| `concepts` | ✅ | 概念标签数组（至少2个） |
| `text` | ✅ | 提炼后的知识单元 |
| `source.chapter_file` | ✅ | 来源章节文件 |
| `source.page_range` | ✅ | 原文页码范围 |
| `confidence` | ✅ | 置信度：high/medium/low/contested |
| `scope_note` | ❌ | 适用范围或限制说明 |
| `fiction_application` | ❌ | 理论→小说的用法桥接 |
| `methodology` | ❌ | 可操作方法步骤（仅适用时填写） |

**domain 域值表**：

| domain 值 | 含义 | 示例书籍 |
|-----------|------|---------|
| `writing_craft` | 写作技法理论 | 麦基、特鲁比、伯罗薇 |
| `political_economy` | 政治经济学 | 《资本论》 |
| `worldbuilding` | 世界观构建 | economic-systems skill |
| `sociology` | 社会学/社会理论 | (计划中) |
| `philosophy` | 哲学 | 哲学参考 |
| `history` | 历史学 | (计划中) |

**领域特有扩展字段**：

部分 domain 可以增加可选扩展字段。扩展字段写在顶层，不嵌套，避免数据层次过深。

| domain | 扩展字段 | 取值 | 说明 |
|--------|---------|------|------|
| `writing_craft` | `applicable_media` | `["film","novel","tv","webnovel","hybrid"]` | 适用媒介 |
| `writing_craft` | `framework_bias` | `string` | 理论框架偏向说明（如"麦基特有理论建构"） |
| `writing_craft` | `certainty` | `"high"\|"medium"\|"debatable"` | (兼容旧字段，优先使用confidence) |

**confidence 判定标准**：

| 等级 | 定义 |
|------|------|
| high | 普遍共识、基本原理、事实性信息 |
| medium | 主流接受但依赖语境，或领域内普遍认可 |
| low | 个人观点、尚待验证、推测性 |
| contested | 学术界有直接争议、多种对立解释 |

**fiction_application 字段指南**：

这是理论→创作的桥接字段，目的是让任何领域的知识都能被小说创作者直接使用。例如：

- 政治经济学条目 → "这个经济规律可以用来设计虚构世界中贸易联盟的兴衰机制"
- 社会学条目 → "这个社会分层理论可为反乌托邦小说的阶级设计提供依据"
- 写作技法条目 → "这个节拍理论可直接用于场景转折设计"

**概念标签体系**（每本书特有 + 通用参考）：

通用写作概念参考：
- 故事结构：structure, plot, three-act, inciting-incident, climax, resolution
- 人物：character, arc, motivation, conflict, dialogue
- 场景：scene, beat, turning-point, value-change
- 类型：genre, convention, audience
- 技法：show-vs-tell, pacing, subtext, voice, POV

政治经济学概念参考（《资本论》等）：
- 商品：commodity, use-value, exchange-value, value-form
- 货币：money, circulation, means-of-payment
- 资本：constant-capital, variable-capital, surplus-value
- 积累：accumulation, primitive-accumulation, reproduction
- 危机：crisis, overproduction, tendency-of-rate-to-fall

**输出**：
- `references/knowledge-base.json`（核心资产）

### 阶段 3：专题深化与案例构建

**目标**：将全书的核心方法提炼为独立专题文档 + 应用案例。

**参考 karpathy-llm-wiki 的三层架构**：
- `references/deep/` 对应 wiki 的概念层（Concept page）
- `references/case-studies/` 对应案例层（Summary page）

**专题文档**：每个核心概念写一个独立 `.md`，包含定义、判定标准、常见错误。

**案例文档**（可选）：将该书方法应用到已知作品的分析。

**输出**：
- `references/deep/*.md`
- `references/case-studies/*.md`

### 阶段 4：技能组装

**目标**：将以上产出物整合为一个可安装的 OpenClaw skill。

**步骤**：
1. 编写 `SKILL.md`
2. 编写 `scripts/query_kb.ps1`
3. 编写 `_meta.json`
4. 整个目录放入 `~/.config/opencode/skills/` 下

## 三、SKILL.md 模板

```markdown
---
name: {book-slug}
description: >
  基于《原著名》的理论框架，对文学创作进行专业分析与指导。
  内置知识库支持从原著中检索精确的理论原文。
---

# {书名} 知识库分析技能 v1

基于《原著名》的专业创作分析系统。

## 核心定位

你是一位熟读《原著名》的创作导师。你拥有从该书提取的结构化知识库，
你的分析精确引用该书的概念、判定标准、常见错误。

## 知识库检索

在分析前，先从知识库中检索原文引用：
- 调用 scripts/query_kb.ps1 查询关键词
- 或直接读取 references/knowledge-base.json
- 将检索到的引用融入诊断

## 分析维度

（根据该书的特色定义分析维度）

### 维度 1：{维度名}
- 加载 references/deep/{topic}.md
- 引用知识库中的相关条目
- 诊断模板：[理论] → [分析] → [建议]

## 工作流

### Step 1: 作品适用性检查
### Step 2: 路由（纯点评/共创修改）
### Step 3: 检索与诊断
### Step 4: 按语气输出
```

## 四、查询脚本模板

参考 mckee 的 `query_mckee_kb.ps1`，各书调整 conceptMap：

```powershell
param(
    [Parameter(Mandatory=$true)]
    [string]$Query,
    [int]$Limit = 5
)

$KbPath = ".\references\knowledge-base.json"
$kb = Get-Content $KbPath -Encoding UTF8 | ConvertFrom-Json

# 概念别名映射（每本书特有）
$conceptMap = @{
    "concept1" = "别名1,别名2"
    "concept2" = "别名1,别名2"
}

$keywords = $Query -split '\s+' | Where-Object { $_.Length -gt 0 }
$scored = New-Object System.Collections.ArrayList

foreach ($entry in $kb) {
    $score = 0
    foreach ($c in $entry.concepts) {
        if ($conceptMap.ContainsKey($c)) {
            $cKeywords = $conceptMap[$c] -split ','
            foreach ($ck in $cKeywords) {
                foreach ($qk in $keywords) {
                    if ($ck.Trim() -match $qk -or $qk -match $ck.Trim()) { $score += 3 }
                }
            }
        }
    }
    foreach ($qk in $keywords) {
        if ($entry.text -match [regex]::Escape($qk.Trim())) { $score += 2 }
    }
    if ($score -gt 0) {
        $null = $scored.Add(@{score=$score; chapter=$entry.chapter; concepts=$entry.concepts -join ','; text=$entry.text})
    }
}

$results = $scored | Sort-Object -Property score -Descending | Select-Object -First $Limit
$results | ForEach-Object { Write-Output "$($_.score) | $($_.chapter) | $($_.concepts) `n$($_.text)`n" }
```

## 五、质量标准

| 检查项 | 标准 | 检查方法 |
|--------|------|----------|
| 知识覆盖率 | 核心概念条目完整覆盖全书 | 对照目录检查 |
| 概念标注准确 | 每条至少 2 个概念标签，语义精确 | 抽查 20% 条目 |
| 方法论可执行 | 方法步骤化，不含糊 | 选取 3 个方法盲测 |
| 检索召回率 | 关键词能召回相关知识 | query 脚本测试 10 组关键词 |
| 原文引用精确 | 知识条目忠实于原著 | 对照原文抽查 |
