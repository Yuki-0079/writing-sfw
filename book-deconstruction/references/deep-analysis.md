# 深度分析规范 v2

> 供子 agent 在接到模式 A Phase 3（专项提炼）任务时读取。定义如何从完整的 knowledge-base.json 中识别核心概念并撰写深度分析文档。

---

## 0. 输入与输出

### 输入
- `wip/{book-slug}/knowledge-base.json`（完整的 KB）
- `wip/{book-slug}/overview.json`（框架坐标 + 候选概念）

### 输出
写入 `wip/{book-slug}/references/deep/{NN}-{concept-slug}.md`，每篇 1 个核心概念。

---

## 1. 核心概念识别方法

扫描完整 KB，按以下维度筛选高频核心概念：

### 维度 1：术语频率
在 KB 中搜索以下指标：
```
- 该概念作为 concepts 标签出现的次数
- 该概念在 text/scope_note 中被交叉引用的次数
```

### 维度 2：理论中枢性
```
- 该概念是否处于本书理论的中心位置？
- 其他概念是否依赖/引用此概念才能理解？
- 去掉此概念后，理论体系是否崩塌？
```

### 维度 3：独创性
```
- 该概念是否是作者特有的理论贡献？
- 或者是重新定义/深化的已有概念？
```

### 选择标准

| 书籍类型 | 深度分析数量 | 选择原则 |
|:---------|:------------|:---------|
| 理论专书（如资本论） | 3-8 篇 | 理论中枢性优先 |
| 技法指南（如写作书） | 3-5 篇 | 独创性 + 实用价值 |
| 文化理论（如启蒙辩证法） | 3-6 篇 | 独创性 + 跨领域可迁移性 |

---

## 2. 输出格式

每篇深度分析文档为独立的 Markdown 文件，命名规则：

```
{两位序号}-{英文短横分隔概念名}.md
```

如：
```
01-commodity-fetishism.md
02-value-form-and-money.md
03-surplus-value.md
```

### 文件模板

```markdown
# {概念名称 — 深度分析}

> 对应条目：{dk-012 ~ dk-016} | 来源：{第一章第4节}

## 定义

{概念的精准定义，3-5句话。必须基于原文，不能自行定义。}

## 判定标准

{判断一个现象/论述是否属于此概念的范畴，给出可操作的标准。}

| 标准 | 说明 |
|------|------|
| {标准1} | {说明} |
| {标准2} | {说明} |

## 子类型（如有）

{概念内部的重要区分。以列表或图表呈现。}

```
{关系图/层级图}
```

## 常见错误

| 错误 | 正确理解 |
|------|---------|
| {常见误解1} | {正确解读} |
| {常见误解2} | {正确解读} |

## {域特定的特殊字段}

### 对于 political_economy 域：创作应用

| 应用场景 | 操作方式 |
|---------|---------|
| {虚构世界场景} | {如何使用此概念} |

### 对于 writing_craft 域：交叉媒介适配

| 媒介 | 适配方式 |
|:-----|:---------|
| film | {电影中的呈现方式} |
| novel | {小说中的呈现方式} |

## 交叉参照

{与已有知识库中其他概念的关联。}

| 相关概念 | 关联方式 | 所属框架 |
|---------|---------|---------|
| {概念A} | {包含/被包含/平行/对立} | {如资本论} |
| {概念B} | {关联说明} | {如马克思政治经济学} |
```

## 3. 字段撰写规范

### 3.1 定义

- 必须基于原文，3-5 句
- 保留核心术语的原文表述
- 必要时附上原文引用，用「」括起

### 3.2 判定标准

- 每个概念给出 2-4 条判定标准
- 标准必须可操作（能用来判断"这是否属于此概念"）
- 用表格呈现

### 3.3 常见错误

- 至少列出 2 个常见错误或误解
- 每个错误后给出正确理解
- 错误必须真实（确实有人会犯的错误），不要编造

### 3.4 交叉参照

- 列出 2-5 个相关概念
- 标注关联方式：包含、被包含、平行、对立、前提、结果
- 标注所属框架（同一本书还是其他框架）

---

## 4. 自检清单

```
□ 定义是否基于原文？没有自行发挥？
□ 定义是否 3-5 句？
□ 判定标准是否 ≥ 2 条？
□ 判定标准是否可操作？
□ 常见错误是否 ≥ 2 条？
□ 常见错误是否真实？
□ 交叉参照是否 ≥ 2 条？
□ 每条的关联关系是否标注清楚？
□ 文件名命名是否符合规范（{NN}-{concept}.md）？
□ 文件头是否标注了对应 KB 条目 ID 和来源？
```

---

## 5. 输出写入规范

- 每篇一个 .md 文件
- 主 agent 负责写入 `wip/{book-slug}/references/deep/`
- 全部完成后，主 agent 更新 overview.json 中的深度文档列表
