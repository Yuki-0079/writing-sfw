# 书籍知识库构建流程规范 v1

> 将理论书籍转化为结构化知识库 skill 的标准化流程。
> 适用于创意写作理论、左翼理论、文化批判、哲学等任何人文学科书籍。

---

## 一、概述

本流程规范定义了将一本书转化为可复用的知识库 skill 的标准操作程序。

### 产出物

每本书最终的 skill 目录结构：

```
{书名}知识库/
├── SKILL.md                          ← 技能主指令文件
├── references/
│   ├── knowledge-base.json           ← 结构化知识库（核心资产）
│   ├── toc-index.json                ← 章节索引
│   ├── chapters/                     ← 原文分章文件（可选）
│   │   ├── 0001.txt
│   │   └── ...
│   └── deep/                         ← 专题深度分析文档
│       ├── 01-concept-a.md
│       └── ...
├── scripts/
│   └── query_kb.ps1                  ← 知识库查询脚本
└── _meta.json
```

### 输入

- 一本书的文本格式（epub / pdf / markdown）

---

## 二、转化流水线：四阶段

### 阶段 1：文本提取与分章

**目标**：将原始书籍转为可处理的纯文本，按自然章节拆分。

**步骤**：
1. epub → 用 parse_epub.py 提取为纯文本
2. PDF → 直接读取或 OCR
3. 清洗：去除页眉页脚、出版信息、索引等非正文内容
4. 按章节拆分文件，命名为 `NNNN.txt`（4位数字编号）
5. 生成 `toc-index.json`：记录每个章节的文件名和标题

### 阶段 2：知识提取与概念标注

**目标**：逐章精读，提取可复用知识点，标注概念标签。

**阅读规范**：在提取前必须加载 `close-reading-protocol` 按逐块精读流程执行。

**方法（拆→归→联→提）**：

对每一章：
1. **拆分**：将章节内容按语义单元切分
2. **归纳**：提取核心论点、方法、案例、定义
3. **联系**：标注与书中其他章节的关联概念
4. **提炼**：将描述性文字转述为可执行的方法论

**知识条目 JSON 格式（Schema v2）**：

```json
{
  "id": "book-slug-001",
  "domain": "writing_craft | political_economy | cultural_theory | ...",
  "chapter": "章节标题",
  "heading": "小节标题",
  "concepts": ["concept1", "concept2"],
  "text": "200-600字，提炼后的知识单元",
  "confidence": "high | medium | low | contested",
  "scope_note": "适用范围和条件",
  "fiction_application": "理论→创作的桥接（可选）"
}
```

### 阶段 3：专题深化

**目标**：将全书的核心方法提炼为独立专题文档。

- 识别 3-8 个核心概念
- 每个概念写一个独立 `.md`，包含定义、判定标准、常见错误、交叉参照

### 阶段 4：技能组装

**目标**：将以上产出物整合为一个可安装的 skill。

1. 编写 `SKILL.md`
2. 编写 `scripts/query_kb.ps1`
3. 编写 `_meta.json`
4. 安装到 `~/.workbuddy/skills/{book-slug}/`

## 三、质量标准

| 检查项 | 标准 | 检查方法 |
|--------|------|----------|
| 知识覆盖率 | 核心概念条目完整覆盖全书 | 对照目录检查 |
| 概念标注准确 | 每条至少 2 个概念标签，语义精确 | 抽查 20% 条目 |
| text 三层结构 | 包含定义+例证+理论定位 | 抽查条目 |
| text 篇幅 | 200-600 字，len() 验证 | 抽查条目 |
| scope_note 边界 | 同时说明适用和不适用场景 | 抽查条目 |
| 检索召回率 | 关键词能召回相关知识 | query 脚本测试 |
| 原文引用精确 | 知识条目忠实于原著 | 对照原文抽查 |
