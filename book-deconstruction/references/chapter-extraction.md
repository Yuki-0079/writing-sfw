# 章节拆解规范 v2

> 供子 agent（general 类型）在接到章节拆解任务时读取。定义输出格式、字段规则、领域扩展、质检标准。

---

## 0. 前置阅读协议

在开始拆解之前，**必须先加载并执行 close-reading-protocol**：

```
1. 将本章全文按自然语义边界分块（每块 ≤ 2000 字）
2. 逐块精读，每块输出摘要 + 关键要素 + 原文摘录
3. 全部读完后再进行条目提取
```

禁止跳读、抽样读、只读首尾。

---

## 1. 输出格式（Schema v2）

每条知识条目为 JSON 对象，写入 `knowledge-base.json`（追加模式）：

```json
{
  "id": "{book-slug}-{NNN}",
  "domain": "political_economy | writing_craft | cultural_theory | ...",
  "chapter": "第X章 章名",
  "heading": "节标题（如无可省略）",
  "concepts": ["概念标签1", "概念标签2"],
  "text": "200-600字，基于原文的准确概括或引用",
  "confidence": "high | medium | low",
  "scope_note": "该条目的适用范围和边界条件",
  "fiction_application": "如何用于创作实践或理论分析"
}
```

### id 编码规则

| 书 | 前缀 |
|:---|:-----|
| 资本论第一卷 | `dk-` (das kapital) |
| 资本论第二卷 | `dk2-` |
| 资本论第三卷 | `dk3-` |
| 共产党宣言 | `cm-` |
| 德意志意识形态 | `di-` |
| 1844手稿 | `em-` |
| 一般规则：`{书前缀}-{三位序号}`，如 `dk-001`, `cm-012` |

## 2. 字段撰写规则

### 2.1 text（核心字段）

| 规则 | 说明 |
|:-----|:------|
| **必须基于原文** | text 必须是原文的准确概括或直接引用，不能是 agent 自己的发挥/总结 |
| **篇幅控制** | **200-600 字（中文）。注意：200字中文比想象中多，写完后必须用len()计数确认。** 过短则信息不足，过长则不利于检索 |
| **三层结构** | 每条text应当包含三层：(1)**定义**——该概念/论点是什么；(2)**例证**——原文中如何展开论述（如有具体案例更好）；(3)**理论定位**——这个概念在全书框架中的位置或意义。切忌只写定义就收笔 |
| **术语保留** | 重要概念必须保留原文术语（如"社会必要劳动时间"不能简化为"劳动时间"） |
| **单条聚焦** | 一条只讲一个概念或一个论点，不要合并多个独立概念 |
| **引用格式** | 直接引用用「」括起，并标注出处 |

**篇幅控制的最佳实践**：写完后用 `len(text)` 确认中文字数。200字中文约等于4-5句完整论述。如果不足200字，往往是因为缺少例证或理论定位。**不要凭感觉判断字数**——200字中文看起来不多但写够需要三层结构都覆盖。|

### 2.2 domain（领域）

| domain 值 | 适用场景 | 说明 |
|:----------|:---------|:-----|
| `political_economy` | 马克思主义政治经济学 | 资本论、政治经济学批判等 |
| `writing_craft` | 创意写作技法 | 麦基、特鲁比、伯罗薇等 |
| `cultural_theory` | 文化理论/批判理论 | 法兰克福学派、文化研究、后殖民理论等 |
| `literary_theory` | 文学理论 | 叙事学、文体学等 |
| `philosophy` | 哲学 | 本体论、认识论、伦理学等 |
| `sociology` | 社会学 | 阶级分析、社会分层等 |
| `political_theory` | 政治理论 | 国家理论、治理理论等 |

### 2.3 concepts（概念标签）

- 每条约 1-4 个概念标签
- 标签必须是**书中明确使用或定义的术语**
- 不要自创标签名
- 使用通用命名，便于跨书检索

### 2.4 confidence（确信度）

| 等级 | 定义 | 示例（political_economy域） | 示例（writing_craft域） |
|:-----|:------|:---------------------------|:-----------------------|
| `high` | 多框架共识，或基本创作/分析原则 | "商品是使用价值和价值的统一体" | "角色需要动机" |
| `medium` | 主流接受但依赖特定前提 | "工资是劳动力价值的转化形式"（基于劳动价值论） | "主角应该是主动的"（小情节例外） |
| `low` | 作者特有理论建构，或学界有争议 | "利润率趋于下降"（历来有争议） | "单一主角是必要的" |

判定规则：
- **介绍历史事实/其他理论家的观点** → `high`
- **作者主流但有个人倾向的论点** → `medium`
- **作者特有的理论建构** → `low`
- **当 domain=political_economy 时，confidence 等效于 certainty**

### 2.5 scope_note（适用范围）

> 该条目成立的条件和边界。**必须同时说明适用场景和不适用/需调整的场景。**

撰写模板（二选一，推荐模板A）：

**模板A（推荐）——直接指明适用范围和限制：**
```
适用于[场景A、场景B]。不适用于[场景C]；在[场景D]中需要调整，因为[原因]。
```

**模板B（适用于历史/语境依赖较强的条目）：**
```
该主张基于[具体前提/历史条件/理论假设]。
在[偏离条件]时需要调整。
```

✅ 正确示例（含适用+不适用）：
```
适用于分析资本主义商品生产社会中的经济循环。
不适用于非商品生产社会（如自给自足经济）。
```

❌ 不合格示例（只说适用不说限制）：
```
适用于分析经济循环。
```
（缺少边界条件 → 无效 scope_note）

❌ 不合格示例（空泛）：
```
该主张有重要意义。
```
（未说明任何边界 → 无效 scope_note）

**实践原则**：写完 scope_note 后，检查是否包含以下关键词之一：`适用于`、`适用`、`不适用`、`需要调整`、`注意`。如果都没有，说明可能缺少边界条件。

### 2.6 fiction_application（创作/分析应用）

> 将理论概念转化为实践工具的桥梁。

撰写模板：
```
在[场景/情境]中，[概念]可用来[做什么]，效果是[什么]。
```

示例（political_economy）：
```
在设计虚构世界的经济体系时，"社会必要劳动时间"概念可用来设定不同种族/阶层的劳动效率差异，
进而影响贸易比价和阶级关系。
```

示例（cultural_theory）：
```
在分析一部电影的文化政治时，"文化霸权"概念可用来解读主流叙事如何塑造观众的"常识"，
以及文本中是否存在抵抗性元素。
```

**禁止**：空泛套话（如"这个概念可以帮助作者写出更好的故事"）

## 3. 领域扩展字段

不同 domain 在通用 schema 基础上可扩展额外字段：

### 3.1 writing_craft（写作技法）扩展

```json
{
  "certainty": "high | medium | debatable",
  "applicable_media": ["film", "novel", "tv", "webnovel", "hybrid"],
  "framework_bias_note": "（仅用于 medium/debatable 条目）该主张基于..."
}
```

| applicable_media | 含义 | 触发词 |
|:-----------------|:-----|:--------|
| `film` | 以电影为基准 | "银幕""观众""2小时""镜头" |
| `tv` | 以电视剧为基准 | "剧集""季""每周""分集" |
| `novel` | 以长篇小说为基准 | "读者""章节""页码" |
| `webnovel` | 以网络连载为基准 | "更新""连载""追读" |
| `hybrid` | 各媒介通用 | 无特定媒介标识 |

### 3.2 political_economy 扩展

基础字段已够用，无需扩展。但 `fiction_application` 必须填写。

### 3.3 cultural_theory / literary_theory 扩展

```json
{
  "critique_application": "该概念可用于分析何种文化/文本现象",
  "related_theorists": ["与本概念相关的其他理论家"]
}
```

## 5. 常见错误（禁止清单）

| 错误 | 说明 | 正确做法 |
|:-----|:------|:---------|
| ❌ text 脱离原文 | 写了一段原文没有的内容，或过度发挥 | text 必须有原文依据 |
| ❌ **text 过短（最常见错误）** | 少于 200 字，往往只写了定义就收笔 | 采用"定义+例证+理论定位"三层结构，写完后用 len() 确认字数 |
| ❌ **text 只有定义没有展开** | 写了一个概念的定义，但没有原文中的例证支撑 | 补充原文引用的具体案例或论述过程 |
| ❌ 概念标签不一致 | 同一概念在不同条目中用了不同标签名 | 全书概念标签名统一 |
| ❌ 多条合并 | 一条里塞了 2-3 个独立论点 | 拆成多条，单条聚焦 |
| ❌ **scope_note 只说适用不说限制（常见）** | 只写了"适用于X"，没写"不适用于Y" | 用模板：适用于[X]，不适用于[Y]；需要同时写适用和不适用 |
| ❌ **scope_note 空泛** | "该主张有重要意义"——没有边界信息 | scope_note 必须包含具体的适用条件词（适用/不适用/注意） |
| ❌ fiction_application 空泛 | "作者可以理解这个" | 必须有具体场景+行动 |
| ❌ 跳读 | 只看开头结尾就提取条目 | 必须先执行 close-reading-protocol |

## 6. 自检清单（输出前执行）

```
□ 是否已执行 close-reading-protocol 逐块精读？
□ 逐块输出是否完整覆盖了本章全部分块？

□ 本条 text 是否基于原文？（不得凭空编造）
□ 本条 text 是否 200-600 字？—— ⚠️ 必须用 len(text) 精确计数，不要凭感觉
   → 常见陷阱：200字中文需要4-5句完整论述，只写定义大概率不足
   → 检查是否包含"定义+例证+理论定位"三层结构
□ 本条领域（domain）是否正确？

□ confidence 的判定是否有依据？
   - high → 基本共识 / 历史事实
   - medium → 主流但有前提依赖
   - low → 作者特有 / 学界争议

□ scope_note 是否包含了边界条件？
   → 同时写了"适用X"和"不适用Y"吗？
   → 是否包含关键词：适用于/不适用/需要调整/注意？
   → 如果没有这些词，说明很可能缺少边界条件

□ fiction_application 是否具体到场景+行动？
   - 格式：在[场景]中，[概念]用来[做什么]
   - 禁止空泛套话

□ 如果是 writing_craft 域：
   - certainty 是否填写？
   - applicable_media 是否基于本章例证判断？
   - framework_bias_note 是否仅用于 medium/debatable？

□ 如果是 cultural_theory 域：
   - critique_application 是否填写？
   - related_theorists 是否填写？
```

## 7. Token 预算控制

| 约束 | 值 |
|:-----|:----|
| 单步输出上限 | ≤ 15k tokens |
| 每条 text | 200-600 字 |
| 每章提取 | 大章（原文 >5000 字）8-15 条，小章 3-8 条 |
| 大章拆分 | 如果本章原文超过 3000 字，可分 2 次 session 处理 |

---

## 8. 输出写入规范

- 提取的条目以 JSON 数组形式返回给主 agent
- 主 agent 负责追加到 `wip/{book-slug}/knowledge-base.json`
- 如果本章很大（预计超过 15 条），返回部分条目，主 agent 分多次派发

```json
[
  {
    "id": "dk-042",
    ...
  },
  {
    "id": "dk-043",
    ...
  }
]
```
