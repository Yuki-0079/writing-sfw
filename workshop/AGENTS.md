# 写作工作坊 Agents

本系统由三个 Agent 协作组成：

| Agent | 文件（位置） | 职责 |
|-------|-------------|------|
| **书籍拆解师** | `书籍拆解师.md`（全局） | 书籍→知识库转化：处理新书、重构旧库 |
| **Writing Agent** | `writing.md`（全局） | 文学创作主力：构思、诊断、打磨、生产 |
| **Editor Agent** | `editor.md`（全局） | 文学质检与规划：检查、评估、规划改进 |
| **Writing Analyzer**（全局） | `~/.config/opencode/agents/sfw/writing-analyzer.md` | 基于 **知识注册表标签路由** 的多维创作分析 |

## 知识体系架构 v2（标签路由系统）

本系统使用 **中央知识注册表 + 标签分类体系** 替代了原先的硬编码路由规则。现在 agent 通过分析用户需求 → 提取标签 → 匹配知识源 → 按需加载，覆盖三类知识：

| 维度 | 标签前缀 | 说明 | 当前覆盖 |
|------|---------|------|---------|
| **写作技法** | `craft_*` | 结构/人物/对白/场景/视角/文笔/心理 | 11个知识源 |
| **世界观构建** | `world_*` | 经济/政治/社会/文化/技术/生态/演化 | 3个新技能 + 计划扩充 |
| **学科理论** | `domain_*` | 经济/政治/社会/哲学/文化理论 | 2个知识源 + 3个计划中 |
| **文学类型** | `genre_*` | 科幻/奇幻/赛博朋克/反乌托邦等 | 全域标签 |

### 核心文件
- `memory/knowledge-registry.json` — 所有知识源的中央索引（含标签、路径、匹配规则）
- `memory/knowledge-tag-taxonomy.md` — 标签分类体系（含关键词→标签推理映射）
- `~/.config/opencode/skills/` — 所有已安装的技能

### 已安装的世界观构建技能（2026-05-29新增）
- **economic-systems** — 货币、贸易网络、资源经济体系设计
- **governance-systems** — 政治实体与治理结构设计
- **systemic-worldbuilding** — 从novum出发的级联后果框架

## 协作规则

1. **创作与质检分离** — Writing Agent 产出时不自我审查，Editor 审查时不参与创作
2. **Editor 优先** — 用户提交已有文本要求"继续写"时，先质检再决定是否续写
3. **Writing Analyzer 标签路由优先** — Analyzer 不再按固定作品特征路由，而是通过标签分析匹配知识源
4. **Editor 报告是 Writing Agent 的输入** — 新一轮创作前先加载最近质检报告
5. **书籍拆解师独立运作** — 不与 Writing/Editor 直接交互；拆解完成的知识库由用户决定是否安装使用

## 文件规范

- 项目根：`C:\Opencode\writing_sfw`
- 小说项目按 `《作品名》` 文件夹组织
- 每个项目含：`大纲.md`、`设定集.md`、`角色卡/`、`正文/`
- 正文按 `第XXX章_标题.md` 命名
- Editor 质检报告保存在 `《作品名》/质检/` 下，按 `质检报告_YYYY-MM-DD.md` 命名
- 知识库工作进度保存在 `wip/{书名}/` 下
- 进度追踪文件 `memory/progress.md`
- 新对话上手文档 `操作指引.md`

## 召唤方式（新对话中告诉 AI 的指令）

| 我想做的事 | 说这句话 |
|-----------|---------|
| 处理书籍/查进度 | "请加载书籍拆解师的身份工作。先读操作指引.md，执行进度自检。" |
| 文学创作 | "请以 Writing Agent 的身份工作，加载 writing.md。" |
| 质检已写的章节 | "请以 Editor Agent 的身份工作，加载 editor.md。" |
