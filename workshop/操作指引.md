## 一、项目概述

这是一个**创意写作辅助系统**建设项目。核心资产包括写作技法知识库 + 世界观构建技能 + 学科理论知识源，通过**标签路由系统**按需匹配。覆盖三类知识：写作技法（craft_*）、世界观构建（world_*）、学科理论（domain_*）。

### 知识体系架构（v2 标签路由系统）

```
用户需求 → 标签分析 → knowledge-registry.json 匹配 → 按需加载技能
```

中央索引：`memory/knowledge-registry.json`（所有知识源的标签、路径、匹配规则）
标签分类：`memory/knowledge-tag-taxonomy.md`（关键词→标签推理映射）

## 二、进度自检流程

启动后，请执行以下自检并向用户汇报结果：

### Step 1：读进度文件

```markdown
读取 memory/progress.md → 了解各书的状态（未开始 / 进行中 / 已完成）
```

### Step 2：扫描书籍源文件

```markdown
扫描 books/ 目录 → 列出所有原始书籍的文件名和格式
```

### Step 3：检查 wip 目录

```markdown
读取 wip/{book-slug}/knowledge-base.json 的前 5 行 → 检查各书的 KB 完成状态
```

### Step 4：检查已安装的 KB skill

```markdown
检查 ~/.config/opencode/skills/ 下的知识库 skill → 列出已完整安装的
```

### Step 5：汇总汇报给用户

```markdown
项目状态摘要：
- 待处理书籍：N 本
  - 《书名1》— 未开始
  - 《书名2》— Phase 2 进行中（第 7/15 章）
- 已有知识库：N 个（mckee-story-analyzer + ...）
- 下一步建议：[根据状态推荐下一步行动]
```

## 三、可用操作模式

### 模式 A：处理新书

加载 `书籍拆解师.md`，以该 agent 身份执行 **模式 A**（四阶段），指定目标书名。

适用场景：
- 有未处理的新书在 `books/` 目录中
- 用户说"把这本拆了"、"做这本的 KB"

### 模式 B：重构旧库

加载 `书籍拆解师.md`，以该 agent 身份执行 **模式 B**，指定目标知识库路径。

适用场景：
- 用户说"升级 mckee 的 KB"、"给现有 KB 补标注"

### 模式 C：跨框架分析（标签路由版）

由 **Writing Analyzer** (`~/.config/opencode/agents/sfw/writing-analyzer.md`) 执行。

**核心流程**：
1. 分析用户需求 → 提取标签（genre_* / world_* / domain_* / craft_*）
2. 查询 `memory/knowledge-registry.json` → 按标签匹配度排序
3. 加载匹配的知识源 → 多视角分析

**适用于**：
- 需要跨写作技法 + 世界构建 + 学科理论的多维创作分析
- 需要设计虚构世界的经济/政治/社会体系
- 需求涉及多个领域交叉（如赛博朋克需要经济+政治+科技+叙事）
- 用户说"帮我分析这个故事"、"看看这个世界观"

### 模式 D：创作与质检

非书籍拆解师职责。由 `writing.md`（Writing Agent）和 `editor.md`（Editor Agent）处理。

## 四、关键参考

### 知识条目 Schema v2（通用领域）

**核心结构**（适用于所有 domain）：

```json
{
  "id": "book-slug-NNN",
  "domain": "writing_craft | political_economy | worldbuilding | ...",

  "chapter": "章节标题",
  "heading": "小节标题",
  "concepts": ["concept1", "concept2"],
  "text": "提炼后的知识单元",

  "source": {
    "chapter_file": "chapters/NNNN.txt",
    "page_range": "45-48"
  },

  "confidence": "high | medium | low | contested",
  "scope_note": "适用范围和条件（可选）",

  "fiction_application": "这个知识在小说/世界观构建中的用法（可选）",

  "methodology": {
    "steps": ["步骤1", "步骤2"],
    "key_points": ["关键点"]
  }
}
```

**领域特有扩展字段**：

| domain | 扩展字段 | 说明 |
|--------|---------|------|
| `writing_craft` | `applicable_media`, `framework_bias`, `certainty（兼容旧字段）` | 媒介适配和框架偏向 |

### confidence 判定标准（通用）

| 等级 | 定义 |
|------|------|
| high | 普遍共识、基本原理、事实性信息 |
| medium | 主流接受但依赖语境 |
| low | 个人观点、尚待验证、推测性 |
| contested | 学术界有直接争议、多种对立解释 |

> 写作技法旧库（mckee 等）仍使用原有的 `certainty`（high/medium/debatable）字段，两者兼容。新库统一使用 `confidence`。

```

   书名.epub / 书名.pdf

wip/{book-slug}/               ← 工作进度目录
   overview.json                ← Phase 1 产出
   knowledge-base.json          ← Phase 2 产出
   kb_checkpoint.json           ← Phase 2 断点续传
   references/deep/*.md         ← Phase 3 产出
   SKILL.md                     ← Phase 4 产出
   scripts/query_kb.ps1         ← Phase 4 产出
   _meta.json                   ← Phase 4 产出

~/.config/opencode/skills/{skill-name}/   ← 最终安装位置
```
